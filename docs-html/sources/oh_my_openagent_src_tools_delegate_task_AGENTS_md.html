OVERVIEW

b004

49 files. The task tool implementation — delegates work to subagents via background or sync sessions. Resolves categories, models, skills, and manages both async and synchronous execution flows. 8+ built-in categories.

b005

TWO EXECUTION MODES

b006
ModeFlowUse Case
Background (run_in_background=true)Launch → BackgroundManager → poll → notify parentExplore, librarian, parallel work
Sync (run_in_background=false)Create session → send prompt → poll until idle → return resultSequential tasks needing immediate result

BACKGROUND EXECUTION

background-task.ts → BackgroundManager.launch() → (async polling) → background-continuation.ts

b016
  1. Check user-defined categories (pluginConfig.categories)

  4. Check model availability → fallback if unavailable

unstable-agent-task.ts marks tasks from categories/agents known to be unstable (e.g., free models). Enables unstableAgentBabysitter hook monitoring.

概覽

b004

49 個檔案。task 工具的實作 - 透過背景或同步工作階段將工作委派給子代理。負責解析類別、模型、技能，並管理非同步與同步的執行流程。內建 8 種以上類別。

b005

兩種執行模式

b006
模式流程使用情境
背景 (run_in_background=true)啟動 → BackgroundManager → 輪詢 → 通知父層探索、librarian、平行工作
同步 (run_in_background=false)建立工作階段 → 傳送提示 → 輪詢直到閒置 → 回傳結果需要立即結果的順序型任務
b007

主要檔案

b008
檔案用途
tools.tscreateDelegateTask() 工廠 - 主要進入點
executor.ts路由到背景或同步執行
types.tsDelegateTaskArgs、DelegateTaskToolOptions、ToolContextWithMetadata
category-resolver.ts將類別名稱對應到 model + config
subagent-resolver.ts將 subagent_type 對應到 agent + model
model-selection.ts檢查模型可用性 + fallback
skill-resolver.ts解析 load_skills[] → 要注入的技能內容
prompt-builder.ts使用技能內容、類別建立 system/user prompt
b009

同步執行鏈

b010
sync-task.ts → sync-session-creator.ts → sync-prompt-sender.ts → sync-session-poller.ts → sync-result-fetcher.ts
b011

每個檔案處理一個步驟。sync-continuation.ts 處理工作階段延續（以 task_id 重新開始）。

b012

背景執行

b013
background-task.ts → BackgroundManager.launch() → (async polling) → background-continuation.ts
b014

background-continuation.ts 處理現有背景任務以 task_id 續接。

b015

類別解析

b016
  1. 檢查使用者定義的類別（pluginConfig.categories）
  2. 回退到內建的 8 種類別
  3. 從類別設定解析 model
  4. 檢查模型可用性 → 若不可用則回退
