OVERVIEW

b005

11 built-in agents. Type enum: src/config/schema/agent-names.ts BuiltinAgentNameSchema. 10 of them register via builtin-agents.ts agentSources record (factory functions). Prometheus is special-cased — it has no createPrometheusAgent factory; instead prometheus-agent-config-builder.ts constructs its config directly during agent-config-handler Phase 3.

b006

All factories follow createXXXAgent(model) → AgentConfig. Each carries a static mode property (AgentFactory type in src/agents/types.ts). Composed via buildAgent().

b021

Model resolution: 4-step pipeline → override → category-default → provider-fallback → system-default. Defined in shared/model-resolution-pipeline.ts.

b022

MODES

b023

Definition (from src/agents/types.ts):

b024
  • primary — respects user's UI-selected model. Used by: sisyphus, hephaestus, atlas, prometheus.
  • subagent — uses own fallback chain, ignores UI selection. Used by: oracle, librarian, explore, multimodal-looker, metis, momus, sisyphus-junior.
  • all — declared in the type for OpenCode compatibility but no built-in agent currently uses it.
b025

CANONICAL ORDER

b026

Sisyphus → Hephaestus → Prometheus → Atlas (primary core agents) then alphabetical for the rest. Enforced by installAgentSortShim() — patches Array.prototype.{toSorted,sort} narrowly when ≥2 canonical core agents are in the array. See src/plugin-handlers/AGENTS.md for the full history.

b027

DYNAMIC PROMPT BUILDER

b028

dynamic-agent-prompt-builder.ts composes per-agent system prompts at runtime by stitching:

  • Core sections (identity, mode, restrictions)
  • Policy sections (citation, verification, anti-patterns)
  • Tool categorization (per-domain tool guidance)
  • Category-skills guide (which skills load with which categories)
b029

This is what the Sisyphus prompt's "AGENTS / CATEGORY + SKILLS" tables come from.

b021

模型解析：4 步驟管線 → override → category-default → provider-fallback → system-default。定義於 shared/model-resolution-pipeline.ts。

b022

模式

b023

定義（來自 src/agents/types.ts）：

b024
  • primary — 尊重使用者在 UI 中選取的模型。用於：sisyphus、hephaestus、atlas、prometheus。
  • subagent — 使用自己的回退鏈，忽略 UI 選擇。用於：oracle、librarian、explore、multimodal-looker、metis、momus、sisyphus-junior。
  • all — 已在型別中宣告以相容 OpenCode，但目前沒有任何內建代理使用它。
b025

標準順序

b026

Sisyphus → Hephaestus → Prometheus → Atlas（主要核心代理）之後，其餘依字母順序排列。由 installAgentSortShim() 強制執行，會在陣列中有 ≥2 個標準核心代理時，限定性地修補 Array.prototype.{toSorted,sort}。完整歷史請見 src/plugin-handlers/AGENTS.md。

b027

動態提示詞建構器

b028

dynamic-agent-prompt-builder.ts 會在執行期間透過拼接以下內容來組成每個代理的系統提示詞：

  • 核心區段（身分、模式、限制）
  • 政策區段（引用、驗證、反模式）
  • 工具分類（各領域的工具指引）
  • 類別與技能指引（各類別會載入哪些技能）
b029

這就是 Sisyphus 提示詞中的 "AGENTS / CATEGORY + SKILLS" 表格的來源。
