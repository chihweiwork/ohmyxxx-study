OVERVIEW

b005

5 TypeScript files plus 3 markdown prompt variants in packages/prompts-core/prompts/prometheus/. Prometheus remains the interview-mode strategic planner, but this directory is now a thin adapter layer. Prompt content lives in packages/prompts-core; src/agents/prometheus/ only loads the right markdown variant and applies runtime tool gating.

b006

This shape follows the package layering refactor in ROADMAP.md: prompts are harness-neutral core assets, while the OpenCode adapter keeps only model routing and runtime integration.

MODEL VARIANT ROUTING

b010

system-prompt.ts exposes getPrometheusPromptSource(model):

b011
  • GPT family models, as detected by isGptModel(model), route to "gpt".
  • Gemini family models, as detected by isGeminiModel(model), route to "gemini".
  • Missing models and all other families route to "default".
b012

getPrometheusPrompt(model, disabledTools) then loads the selected markdown through loadPromptSync({ source: prometheusPromptVariants[variant], name: "prometheus", variant }) and returns the loaded body.

b013

DISABLED TOOL HANDLING

b014

Prometheus normally includes Question({ ... }) examples because interview mode uses the Question tool to clarify scope. When the runtime passes disabledTools containing "question", getPrometheusPrompt() strips fenced TypeScript Question({ ... }) examples with QUESTION_TOOL_BLOCK_RE before returning the prompt.

b015

This filtering is runtime adapter behavior. Do not duplicate stripped markdown variants in packages/prompts-core; keep one source of truth per model family and let system-prompt.ts remove Question examples only when the tool is disabled.

b016

概覽

b005

在 packages/prompts-core/prompts/prometheus/ 中有 5 個 TypeScript 檔案加上 3 個 Markdown prompt 變體。Prometheus 仍然是訪談模式的策略規劃器，但這個目錄現在只是薄薄的一層適配器。Prompt 內容放在 packages/prompts-core；src/agents/prometheus/ 只負責載入正確的 Markdown 變體並套用執行期工具閘門。

b006

這種結構遵循 ROADMAP.md 中的套件分層重構：prompts 是與 harness 無關的核心資產，而 OpenCode adapter 只保留模型路由與執行期整合。

模型變體路由

b010

system-prompt.ts 會提供 getPrometheusPromptSource(model)：

b011
  • 由 isGptModel(model) 判定的 GPT 家族模型會路由到 "gpt"。
  • 由 isGeminiModel(model) 判定的 Gemini 家族模型會路由到 "gemini"。
  • 缺少模型以及所有其他家族都會路由到 "default"。
b012

getPrometheusPrompt(model, disabledTools) 接著透過 loadPromptSync({ source: prometheusPromptVariants[variant], name: "prometheus", variant }) 載入所選的 markdown，並回傳載入後的內容。

b013

已停用工具處理

b014

Prometheus 通常會包含 Question({ ... }) 範例，因為訪談模式會使用 Question 工具來釐清範圍。當執行期傳入的 disabledTools 包含 "question" 時，getPrometheusPrompt() 會在回傳 prompt 之前，使用 QUESTION_TOOL_BLOCK_RE 移除 fenced TypeScript Question({ ... }) 範例。

b015

這個過濾是執行期適配器行為。不要在 packages/prompts-core 中重複建立已移除內容的 markdown 變體；每個模型家族只保留一份單一真實來源，並讓 system-prompt.ts 只在工具被停用時移除 Question 範例。

b016

b017

  • 禁止路徑：src/、package.json、設定檔
  • 規劃前必須先探索 codebase（絕不可盲目規劃）
  • 計畫會儲存到 .omo/plans/
  • 需要「使用者手動測試」的驗收標準是禁止的
  • Prompt 的修改應放在 packages/prompts-core/prompts/prometheus/，而不是 TypeScript 區段檔案中
