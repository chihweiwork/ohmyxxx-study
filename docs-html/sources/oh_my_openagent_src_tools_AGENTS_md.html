OVERVIEW

b004

Tools registered via createToolRegistry() in src/plugin/. Native tools are factory-based (createXXXTool) except interactive_bash (ToolDefinition). LSP and AST-grep tools are no longer native src/tools/ implementations; they are served by Tier-1 built-in MCPs lsp and ast_grep and keep the same exposed names (lsp_diagnostics, ast_grep_search, etc.).

b019

ADDING A NEW TOOL

b020
  1. Create src/tools/{name}/index.ts with factory createXXXTool
  2. Add types.ts for parameter Zod schemas
  3. Add tools.ts (or single index.ts) for implementation
  4. Export factory from src/tools/index.ts
  5. Register in src/plugin/tool-registry.ts:
  • Always-on: spread into allTools directly
  • Conditional: build a Record<string, ToolDefinition> and gate-spread
  1. If the tool needs disabling, ensure it appears in filterDisabledTools allow-list (its name will be matched against disabled_tools)

新增工具

b020
  1. 建立 src/tools/{name}/index.ts，其中包含 factory createXXXTool
  2. 新增 types.ts 以定義參數的 Zod schemas
  3. 新增 tools.ts（或單一的 index.ts）作為實作
  4. 從 src/tools/index.ts 匯出 factory

  • 一直啟用：直接展開到 allTools
  • 條件式：建立 Record<string, ToolDefinition> 並依條件展開
